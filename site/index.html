Build and evaluate agentic systems

An agentic system is software that uses an LLM to plan, execute, and adapt across multiple steps. At its core, it's a framework — LangChain, Pydantic AI, OpenAI Agents SDK, or your own code — wrapping orchestration logic around one or more LLM endpoints. The framework manages prompts, tools, memory, and routing; the LLM handles reasoning and generation.

These systems range from deterministic workflows (prompt chains, RAG pipelines) to fully autonomous agents that decide their own next steps. Domino supports both with the same instrumentation and evaluation infrastructure.

The trace: Domino's core abstraction

A trace captures the full execution path of an agentic system: every LLM call, tool invocation, decision point, and intermediate output, along with token usage, latency, and cost at each step. One decorator (@add_tracing) instruments your code, and the same trace data powers every phase of the lifecycle below.

Domino builds on MLflow's tracing infrastructure and integrates it into the broader ML lifecycle — connecting traces to project structure, governance workflows, deployment infrastructure, and production monitoring. This works whether your system is a RAG pipeline, a prompt chain with validation gates, or a fully autonomous multi-agent system.

Lifecycle

Develop

Instrument & capture traces

Add @add_tracing to your functions. Works with LangChain, OpenAI, Pydantic AI, LlamaIndex, or standalone code. Every call — including downstream tool and agent calls — is recorded.
Experiment

Evaluate & compare

Attach evaluations to traces: programmatic metrics, manual SME labels, or ad-hoc batch runs. Compare configurations side by side with trace diffs to see exactly where changes helped or hurt.
Deploy

Launch to production

Deploy your best configuration. Domino continues collecting traces from real user interactions automatically — no instrumentation changes needed between development and production.
Monitor

Track & iterate

Track performance with production traces. Run evaluations continuously and iterate when accuracy drops, latency spikes, or costs increase. The same trace data powers debugging and audit.

These capabilities live inside your Domino project, making it straightforward for colleagues to review, share, and extend your work.
