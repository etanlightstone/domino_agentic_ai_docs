Deploy agentic systems

After experimenting and identifying a promising configuration, deploy your agentic system to production. Agents use the same infrastructure as Domino Apps with configurations tailored for agentic workloads.

The same instrumentation you used during experimentation continues to work in production, automatically collecting traces from real user interactions.

Prerequisites

  • Project files containing your agent code (you can optionally deploy from an experiment with collected traces)
  • The Domino logging SDK installed in your environment to enable production monitoring
  • Project Collaborator permissions to deploy and edit agents

Step 1: Deploy an agent

You can deploy an agent in two ways:

After comparing evaluation runs and identifying a configuration you want to deploy:

  1. In your project, from the left navigation, click Experiments.
  2. Choose the experiment with the agent configurations you're evaluating.
  3. Select the run you want to launch.
  4. Click … on the right to launch the run.

A deployment wizard opens, similar to the Apps deployment flow. The wizard pre-fills the code commit from the selected run and prompts you to specify the entrypoint for your agent.

Deploy directly from the Apps view in your project. This follows the standard Publish and share an App workflow, with configuration options specific to agentic workloads.

Step 2: Collect production traces

Production traces are collected automatically using the same @add_tracing decorator or init_tracing() function you used during experimentation. No additional instrumentation is needed.

Production initialization: In production mode, init_tracing() requires the environment variables DOMINO_AGENT_IS_PROD and DOMINO_APP_ID to be set. These are configured automatically when deploying through the Domino UI. Call init_tracing() before your app starts to begin logging traces.

from domino.agents.tracing import init_tracing

# Call before your app starts serving requests
init_tracing(autolog_frameworks=["langchain"])

Production traces appear in the agent's Performance tab and include the same details captured during development.
