Set up LLM access

Your agentic system needs access to one or more LLM endpoints. Before you start developing, decide how your agent will reach the models it depends on. Domino supports two approaches: connecting to an external LLM provider or hosting a model inside Domino.

Option A: Connect to an external LLM provider

Most teams already have access to a managed LLM service. Your agent code calls the provider's API directly — Domino runs the agent, and the LLM inference happens outside Domino.

How to configure access in Domino

  1. Install the provider's SDK in your Domino environment. Add the package to your environment's Dockerfile instructions (e.g., RUN pip install openai) or install it in your workspace.
  2. Store API keys as Domino environment variables. In your project settings, add environment variables for your credentials. These are injected into workspaces and Jobs automatically — your code reads them at runtime via os.environ. This keeps secrets out of your code and version control.
  3. Use the provider's SDK in your agent code the same way you would locally. The provider handles model hosting, scaling, and versioning.
import os
from openai import OpenAI

# API key is read from a Domino environment variable
client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "Hello"}]
)

Swapping models during experimentation. One advantage of external providers is that switching models is a configuration change, not an infrastructure change. When you iterate on your agent during development, you can swap between models (e.g., GPT-4o vs. Claude 3.5) by changing a value in your YAML agent config — no redeployment needed.

Option B: Host a model in Domino

If you need to run your own model — for data residency, cost control, or to use a fine-tuned model — Domino can host it as a Model API. Your agent code calls the Domino-hosted endpoint the same way it would call any other API.

See Register and deploy LLMs for the full setup guide, including:

  • Registering a model from Hugging Face or a custom checkpoint
  • Deploying it as a Domino Model API endpoint
  • Configuring GPU hardware and scaling

Mixing approaches

Many production agents use both: an external frontier model for primary reasoning and a Domino-hosted model for specialized tasks (e.g., a fine-tuned classifier or embedding model). The instrumentation in the Develop step captures traces from all LLM calls regardless of where the model is hosted.
