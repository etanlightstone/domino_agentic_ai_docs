Experiment tracking and traces with agents

After instrumenting your agentic system, run experiments to test different configurations and evaluate quality. Domino captures traces from each run and lets you attach evaluations, compare results, and export findings.

View traces

Once you have logged at least one run with traces, view and analyze them in the Experiments tab:

  1. Find and click the experiment you want to review.
  2. Open a run and select the Traces tab.

When reviewing traces you can:

  • See evaluations logged in code
  • Add metrics (float values) or labels (string values) by clicking Add Metric/Label
  • Click any metric or label cell to open the detailed trace view

Log evaluations

Log evaluations inline during execution or ad-hoc after traces are generated. The Domino instrumentation examples repository has working examples for both approaches.

Pass an evaluator argument in the decorator to capture inputs and outputs of the decorated function. An evaluator is a function that receives the span and returns a dictionary of evaluation results.

@add_tracing(
    name="prioritize_ticket",
    autolog_frameworks=["openai"],
    evaluator=judge_response  # Span-level evaluator
)
def run_agent(...):
    ...

You can also pass a trace_evaluator to evaluate the entire trace (not just individual spans). This is valuable for end-to-end quality checks. The trace evaluator fires when the trace was both started and finished by the decorated function.

@add_tracing(
    name="prioritize_ticket",
    autolog_frameworks=["openai"],
    evaluator=judge_response,           # Span-level
    trace_evaluator=check_end_to_end    # Trace-level
)
def run_agent(...):
    ...

Run evaluations after traces have been collected. Use search_traces() to find traces from a run, then log evaluation results against them. This is useful for batch evaluation with SME review or automated scoring pipelines.

from domino.agents.tracing import search_traces

response = search_traces(run_id="your-run-id", trace_name="prioritize_ticket")
for trace in response.data:
    # Evaluate each trace and log results
    ...

Compare traces

Compare runs to see how different configurations perform on the same inputs:

  1. From the experiment view, select two to four runs and click Compare.
  2. Open the Traces comparison view in the Experiment Manager.
  3. Review results across runs to spot patterns. Click any metric or label cell to open the detailed trace view.

Export results

Export data in two formats:

  • CSV — Open the experiment and click the three dots in the upper right.
  • PDF report — Select two to four runs to compare and click the three dots in the upper right.
